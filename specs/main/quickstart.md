# Quickstart Guide: Setting up the Docusaurus Book Project

**Date**: 2025-12-08

This guide provides the steps to set up the Docusaurus project locally.

## Prerequisites

- **Node.js**: Version 18.x or later.
- **npm** or **yarn**: Package manager.

## 1. Initialize the Docusaurus Project

First, create a new Docusaurus site in a `book` directory.

```bash
npx create-docusaurus@latest book classic
```

This command will scaffold a new Docusaurus project in a directory named `book`.

## 2. Project Structure Setup

Navigate into the new directory:
```bash
cd book
```

The default template comes with placeholder content. We need to clear it and create our book's structure.

1.  **Clear existing docs**:
    ```bash
    rm -r docs/*
    ```

2.  **Create module directories**:
    ```bash
    mkdir -p docs/01-ros docs/02-digital-twin docs/03-ai-robot-brain docs/04-vla docs/05-why-physical-ai docs/06-learning-outcomes docs/07-weekly-breakdown docs/08-assessments docs/09-hardware-requirements
    ```

3.  **Create an `intro.md` file**:
    This will be the main landing page for your docs.
    ```bash
    echo "# Introduction" > docs/intro.md
    ```

## 3. Configure Docusaurus

1.  **`docusaurus.config.js`**:
    Update this file with your book's information and GitHub Pages deployment settings.

    ```javascript
    // docusaurus.config.js
    module.exports = {
      title: 'Physical AI & Humanoid Robotics',
      tagline: 'An Open Source Book',
      url: 'https://<YOUR-USERNAME>.github.io',
      baseUrl: '/<YOUR-REPO-NAME>/',
      onBrokenLinks: 'throw',
      onBrokenMarkdownLinks: 'warn',
      favicon: 'img/favicon.ico',
      organizationName: '<YOUR-USERNAME>', // Usually your GitHub org/user name.
      projectName: '<YOUR-REPO-NAME>', // Usually your repo name.

      // ... other settings
    };
    ```

2.  **`sidebars.js`**:
    Configure the sidebar to be auto-generated from the `docs` directory.

    ```javascript
    // sidebars.js
    /** @type {import('@docusaurus/plugin-content-docs').SidebarsConfig} */
    const sidebars = {
      bookSidebar: [{type: 'autogenerated', dirName: '.'}],
    };

    module.exports = sidebars;
    ```

## 4. Run the Development Server

Start the local development server to see your site in action.

```bash
npm run start
```

This will open a browser window with your Docusaurus site. As you make changes to the Markdown files, the site will update automatically.

## 5. Add Content

You can now start adding content by creating `index.md` files within each module directory and other `.md` files for sub-sections. Remember to add `_category_.json` to each module directory to set the label and position for the sidebar.
